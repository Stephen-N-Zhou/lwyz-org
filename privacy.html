Privacy Policy – Gmail Attachment Downloader (Python Script)

Effective date: October 19, 2025

This Python script connects to the Gmail API with gmail.readonly access to identify and download email attachments. All attachments and message data are processed and saved locally on your computer. No data is uploaded, shared, or transmitted to any external server or third party.

1. Overview

The script enables users to download attachments from their own Gmail account using Google’s official Gmail API. It does not modify, send, or delete any emails and does not communicate with any online service other than the Gmail API.

2. Data Accessed

  • Email metadata (e.g., sender, subject, date)
  • Email message content, only as required to locate attachments
  • Email attachments (e.g., PDFs, images, or other file types)

No other Google data or personal information is accessed.

3. API Scope Used

  • https://www.googleapis.com/auth/gmail.readonly – used to read messages and download attachments from the user’s mailbox.

No other scopes (such as modify, send, or full access) are requested or used.

4. How Data Is Used and Stored

  • Local processing only: All Gmail data and attachments are handled on your own device.
  • Attachment saving: When attachments are downloaded, they are saved to a local folder (e.g., attachments/) that remains under your control.
  • No external sharing: Attachments and email data are never uploaded to any external database, server, or cloud service.
  • Token storage: OAuth credentials (e.g., token.json) are stored locally for user convenience; deleting this file revokes access immediately.

5. User Control and Revocation

  • You may delete downloaded attachments or generated files at any time.
  • You may revoke the script’s access to your Gmail account at any time by visiting: https://myaccount.google.com/permissions.

6. Google API Limited Use

This application’s use of information received from Gmail APIs will adhere to the Google API Services User Data Policy, including the Limited Use requirements. The script does not transfer Gmail data to others except as necessary to provide or improve user-facing functionality, comply with applicable law, or as part of a merger/acquisition.

7. Security

  • All processing occurs locally on your device; no remote servers are involved.
  • Users are responsible for securing their local files and folders where attachments are stored.

8. Changes to This Policy

This policy may be updated periodically. Any material changes will be reflected by updating the effective date above.
