Privacy Policy – Gmail Automation Script

Effective date: October 19, 2025

Contact: iambosszhou@gmail.com

This local Python automation script connects to the Gmail API using the gmail.readonly scope. Its primary purpose is to perform email-based automation and analysis tasks, including downloading specific attachments needed for processing. All Gmail data and attachments are handled and stored locally on the user’s device. No data is transmitted, shared, or uploaded to any external server.

1. Overview

The script allows a user to analyze and automate workflows based on their own Gmail account data. For example, it may read certain messages and save related attachments for automated processing. It does not send, delete, or modify any emails.

2. Data Accessed

  • Email metadata (e.g., sender, subject, date)
  • Email content (only to identify automation-related messages)
  • Email attachments (e.g., PDFs, CSVs, or documents used for analysis)

3. API Scope Used

  • https://www.googleapis.com/auth/gmail.readonly – used to read emails and download attachments for local automation or data analysis.

No other Gmail API scopes are requested or used.

4. Data Usage and Storage

  • Local processing only: All Gmail data is processed directly on the user’s computer.
  • Attachment saving: Attachments are downloaded only when relevant to the automation task and are saved in a local folder (e.g., attachments/).
  • No external transmission: The script does not upload, send, or share Gmail data or attachments to any third party or online service.
  • OAuth credentials: Access tokens (e.g., token.json) are stored locally to maintain authorization. Users can delete this file at any time to revoke access.

5. User Control and Revocation

  • Users can delete any locally saved attachments or data outputs at any time.
  • Access to Gmail can be revoked by visiting: https://myaccount.google.com/permissions.

6. Google API Limited Use

This application’s use of information received from Gmail APIs complies with the Google API Services User Data Policy, including the Limited Use requirements. Data retrieved via the Gmail API is not transferred to others except as necessary to provide or improve user-facing functionality, comply with the law, or as part of a merger/acquisition.

7. Security

  • All data remains on the user’s device; no external servers are used.
  • Users are responsible for protecting their local storage environment.

8. Changes to This Policy

Any updates or changes will be reflected by updating the effective date above.
